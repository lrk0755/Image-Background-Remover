## 五、API 对接

### 5.1 Clipdrop API 接入

**接口地址**: `https://clipdrop-api.co/remove-background/v1`  
**请求方式**: POST  
**请求头**:
```
x-api-key: your_api_key
```

**请求体**: 图片二进制数据

**响应**: 返回处理后的图片二进制

### 5.2 后端代码示例 (含隐私保护)

```javascript
// routes/api.js
const express = require('express');
const router = express.Router();
const axios = require('axios');
const FormData = require('form-data');

router.post('/remove-bg', async (req, res) => {
  let imageBuffer = null;
  
  try {
    // 1. 图片直接在内存中处理，不写入磁盘
    imageBuffer = req.body;
    
    const form = new FormData();
    form.append('image_file', imageBuffer, {
      filename: 'image.png',
      contentType: 'image/png'
    });

    const response = await axios.post(
      'https://clipdrop-api.co/remove-background/v1',
      form,
      {
        headers: {
          ...form.getHeaders(),
          'x-api-key': process.env.CLIPDROP_API_KEY
        },
        responseType: 'arraybuffer',
        timeout: 30000
      }
    );

    // 2. 清理原始图片内存
    imageBuffer = null;

    // 3. 返回结果
    res.set('Content-Type', 'image/png');
    res.send(response.data);
    
    // 4. 处理完成后清理内存
    response.data = null;
    
  } catch (error) {
    // 异常时也确保清理
    imageBuffer = null;
    res.status(500).json({ error: '处理失败，请重试' });
  }
});

module.exports = router;
```

## 七、隐私保护方案 (重点优化)

### 7.1 隐私顾虑分析

用户上传照片可能存在的隐私担忧：
- 图片是否会被永久保存？
- 图片是否会用于AI训练？
- 传输过程是否安全？

### 7.2 解决方案

| 措施 | 说明 | 优先级 |
|------|------|--------|
| **不留存原始图片** | 处理完成后立即删除，不写入磁盘 | P0 |
| **内存中处理** | 使用 Buffer 而非文件存储 | P0 |
| **HTTPS 传输** | 全站强制 HTTPS 加密 | P0 |
| **不用于AI训练** | 明确声明条款，用户可选 | P1 |
| **客户端预处理** | 考虑浏览器端方案 | P1 |
| **隐私政策页面** | 清晰告知数据处理方式 | P1 |

### 7.3 隐私架构图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   用户前端   │ ──▶ │   后端 API   │ ──▶ │  Clipdrop   │
│  (浏览器)    │     │  (内存处理)  │     │    API      │
└─────────────┘     └─────────────┘     └─────────────┘
       │                   │                   │
       │              处理完成                  │
       │              立即释放                   │
       │                   │                   │
       ▼                   ▼                   ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  不存储      │     │  Buffer     │     │ 不存储      │
│  浏览器内存  │     │  立即释放    │     │ 仅处理      │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 7.4 可选：纯前端方案 (完全隐私)

如果用户对隐私要求极高，可考虑 **纯前端处理**：

| 方案 | 效果 | 性能 | 隐私 |
|------|------|------|------|
| TensorFlow.js (U2-Net) | ⭐⭐⭐ | 需加载模型 | ✅ 完全本地 |
| imgly-background-removal | ⭐⭐⭐⭐ | 中等 | ✅ 完全本地 |

**推荐**: 使用 **imgly-background-removal** (开源，浏览器端运行)

```javascript
// 纯前端示例 - 完全不上传图片
import { removeBackground } from "@imgly/background-removal";

// 用户选择文件后
const fileInput = document.getElementById('fileInput');
fileInput.addEventListener('change', async (e) => {
  const file = e.target.files[0];
  
  // 纯浏览器处理，不上传任何图片到服务器
  const resultBlob = await removeBackground(file);
  
  // 生成下载链接
  const url = URL.createObjectURL(resultBlob);
  // 用户下载...
});
```

**优势**: 
- ✅ 图片完全在本地处理，不上传
- ✅ 无需后端服务器
- ✅ 零服务器成本

**劣势**:
- ⏳ 首次加载需下载模型 (约 100MB)
- 💻 设备性能要求较高
